# Unity 기반 구독자 참여형 3D 게임쇼 MVP 기획·개발 계획서
## 0. 프로젝트 한 줄 정의

구독자 참가자들이 3D 가상 공간 안에서 미션, 포인트, 상점, 아이템, 배신, 추리, 추격을 경험하고, 운영자가 실시간으로 판을 진행하며 이를 유튜브 콘텐츠로 제작하는 참가형 게임쇼 플랫폼.

## 1. 프로젝트 목표
### 1.1 최종 목표
10~20명의 참가자가 온라인으로 접속한다.
운영자는 게임 마스터이자 상점 관리자, 심판, 방송 진행자 역할을 한다.
참가자는 미션을 수행해 포인트를 획득한다.
포인트로 상점에서 아이템을 구매한다.
아이템은 추리, 도망, 속임수, 협상, 배신, 역전 상황을 만든다.
게임 장면은 운영자 시점, 관전 카메라, 참가자 시점으로 녹화된다.
유튜브 본편, 쇼츠, 라이브, 글로벌 자막/통역 콘텐츠로 재활용된다.
장기적으로는 다른 크리에이터도 구매해 운영할 수 있는 게임쇼 패키지/플랫폼으로 확장한다.
### 1.2 1차 MVP 목표

처음부터 완성형 오픈월드를 만들지 않는다.

1차 MVP는 아래 기능만 목표로 한다.

참가자 4~8명 접속
작은 3D 맵 1개
운영자 1명 접속
제한시간 30~40분
팀 구분
포인트 획득
상점 이용
아이템 3개 사용
운영자 관전 모드
기본 점수판/상태창
OBS 녹화 가능
## 2. 게임 콘셉트
### 2.1 기본 콘셉트

참가자들은 제한시간 안에 맵 안의 미션을 수행하고, 포인트를 모아 아이템을 구매한다. 아이템을 활용해 다른 플레이어를 속이거나, 추격하거나, 방해하거나, 협상하면서 최종 목표를 달성한다.

운영자는 단순 관전자가 아니라 게임의 핵심 장치다.

### 2.2 첫 번째 게임 모드

## 3. 핵심 재미 요소
### 3.1 참가자 재미
내가 직접 게임쇼에 참여한다.
미션을 성공하면 포인트를 얻는다.
포인트로 변칙적인 아이템을 산다.
다른 사람을 속이거나, 협상하거나, 배신할 수 있다.
우승, MVP, 인기상, 명예의 전당 같은 보상을 받을 수 있다.
### 3.2 시청자 재미
누가 배신자인지 추리한다.
참가자의 거짓말과 배신을 본다.
아이템 사용으로 생기는 반전을 본다.
운영자가 판을 흔드는 장면을 본다.
다음 회차 참가 욕구를 느낀다.
### 3.3 운영자 재미
게임 마스터로서 판을 설계하고 조정한다.
상점 관리자처럼 참가자와 거래한다.
긴급 이벤트로 게임 흐름을 조절한다.
방송에 필요한 장면을 의도적으로 만든다.

상점 UI가 열린다.

모든 구매 내역은 로그에 저장한다.

선택지 A: Netcode for GameObjects

## 15. AI 코딩 작업 방식
### 15.1 기본 원칙

AI에게 “게임 전체를 만들어줘”라고 하지 않는다.

항상 아래 형식으로 지시한다.

현재 목표
현재 구현된 기능
이번에 만들 기능 1개
Unity 버전
사용하는 네트워크 프레임워크
원하는 완료 기준
에러가 나면 에러 전문 붙여넣기
### 15.2 좋은 프롬프트 예시
플레이어 이동

“Unity에서 3D 캐릭터 이동을 만들고 싶다. 초보자 기준으로 CharacterController를 사용해서 WASD 이동, 마우스 회전, Shift 달리기를 구현하는 C# 스크립트를 작성해줘. 어떤 GameObject에 붙여야 하는지도 단계별로 설명해줘.”

포인트 시스템

“Unity C#에서 PlayerData에 points 값을 저장하고, 미션 성공 시 AddPoints(int amount)를 호출해 포인트를 증가시키는 구조를 만들고 싶다. 나중에 멀티플레이 서버 권위 구조로 바꿀 예정이므로, 지금은 싱글플레이 버전과 서버 버전으로 확장하기 쉬운 구조로 작성해줘.”

상점 시스템

“Unity에서 상점 UI를 만들고 싶다. 플레이어가 ShopTrigger에 들어가면 상점 패널이 열리고, 보유 포인트가 충분하면 아이템을 구매해 인벤토리에 추가하는 구조를 만들어줘. ItemData를 ScriptableObject로 관리하는 방식으로 설명해줘.”

도플갱어 시약

“Unity 멀티플레이 게임에서 플레이어가 다른 플레이어의 외형으로 5분간 변신하는 아이템을 만들고 싶다. 서버가 변신을 승인하고 모든 클라이언트에 외형 변경이 동기화되어야 한다. 먼저 싱글플레이 구조로 만들고, 이후 Netcode for GameObjects로 확장할 때 필요한 포인트를 설명해줘.”

## 23. 요약

이 프로젝트는 게임 개발 프로젝트이면서 동시에 유튜브 예능 제작 시스템이다.

Unity로 시작할 경우 가장 중요한 전략은 다음이다.

크게 만들지 않는다.
작은 게임쇼 세트장부터 만든다.
아이템은 3개만 만든다.
운영자 기능을 빨리 만든다.

유튜브 반응을 보고 확장한다.

첫 번째 성공 기준은 게임 출시가 아니다.

첫 번째 성공 기준은 “파일럿 영상 1편을 만들 수 있는 플레이 가능한 세션”이다.



정리해뒀어, 대한님.
옆 캔버스에 **「Unity 기반 구독자 참여형 3D 게임쇼 MVP 기획·개발 계획서」**로 만들어놨어.

핵심은 이렇게 잡았어.

**첫 목표는 완성형 오픈월드가 아니라, “8명이 접속해서 운영자가 포인트/아이템을 주고, OBS로 촬영 가능한 작은 게임쇼 세트장”을 만드는 것**이야.

예를 들면 Unity 설치 후 프로젝트 생성, 플레이어 이동, 포인트 UI 만들기까지 바로 따라 하는 식으로.
