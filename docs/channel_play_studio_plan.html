Channel Play Production OS

AI로 Unity 게임을 만들기 위한 제작 운영툴

`Channel Play Studio`는 Unity 구현, Obsidian 지식베이스, AI 에이전트, Blender/2D-to-3D 에셋 파이프라인, 캡처 기반 피드백 루프를 하나로 묶는 프로젝트 전용 제작 환경이다.

Mac Studio Unity Editor, OBS, Blender, GitHub, 구현 본진
gdx1 SSH 인증 복구 후 서버·봇·soak test 워커
Obsidian 디자인, 세션, 피드백, 에셋 브리프의 기억 저장소
channelctl 모든 작업의 단일 명령 진입점

캡처 기반 피드백 루프

사용자가 Unity에서 직접 보고, 캡처/주석을 남기면 AI가 수정 작업으로 변환한다.

1. Play

Unity에서 `School_MVP` 플레이 또는 씬 확인

→

2. Capture

`channelctl capture screen`으로 스크린샷 저장

→

3. Annotate

주석/수정 요청을 feedback record에 기록

4. Interpret

Codex/Claude가 피드백을 작업 단위로 해석

→

5. Implement

Unity 파일 수정, 컴파일/씬 검증

→

6. Verify

before/after 캡처와 세션 요약으로 종료

AI 에셋 파이프라인

2D 이미지, Pixel3D/Meshy류 3D 생성, Blender cleanup, Unity import를 하나의 상태 기계로 관리한다.

01

Brief

Obsidian에 용도, 크기, 방송 가시성, 스타일 정의

02

2D Concept

참조 이미지 또는 AI 2D 컨셉 생성

03

3D Generate

Pixel3D/Meshy류 도구로 초안 mesh 생성

04

Blender Cleanup

스케일, 피벗, 메시, 머티리얼, 콜라이더 후보 정리

05

Unity Import

Unity-ready FBX/GLB, prefab, material 생성

06

Scene Test

`School_MVP`에서 배치, 프레임, 가독성 확인

07

Accept

accepted/rework/rejected 상태로 기록

AI 에이전트 역할 분리

핵심 원칙: 한 브랜치에서는 한 writer만. 나머지는 리뷰/조사/시각 점검/장기 작업으로 분리.

Codex

Claude

Hermes

Agent Company Layer

역할표를 넘어서, 공통 기억과 작업 계약을 가진 제작 회사 구조로 AI들을 운영한다.

상세 계획
Memory

Shared Memory

Obsidian, memory/company, runs, reviews, asset_pipeline이 공통 기억과 증거 저장소가 된다.

Orchestrator

Chief Orchestrator AI

요청을 분해하고, 필요한 에이전트만 호출하며, lock과 검증을 통제한다.

Production Team

Specialist Agents

Unity, 에셋, Blender, 사운드, QA, gdx1, 메모리 관리 역할이 분리된다.

intake 사용자 요청을 목표, 범위, 기대 산출물, 검증 방법으로 변환 required
brief 공통 기억에서 현재 맥락을 뽑아 모든 에이전트가 같은 출발점에서 작업 required
assign 작업별 writer/reviewer/specialist를 최소 인원으로 선정 mvp
lock Unity scene, prefab, script system, asset folder를 path 단위로 충돌 방지 mvp
verify Unity compile, screenshot, playtest, Blender report, gdx1 log 등 증거 없이는 완료 처리 금지 required

`channelctl` 명령 체계

처음부터 거대한 앱을 만들지 않고, 반복 작업을 안정적인 CLI로 묶는다.

status Git, Unity, gdx1, agents, latest run 상태 요약 first
open Unity, Obsidian, dashboard 바로 열기 first
capture 스크린샷/OBS 캡처 후 reviews 폴더에 저장 first
feedback 캡처 기반 수정 요청 템플릿 생성 first
unity compile, scene open, screenshot, build checks next
asset brief, generated_3d, blender cleanup, unity_ready 상태 관리 next
company brief, plan, assign, lock, report, memory sync로 agent company 운영 next
gdx probe, sync, headless server, bot test. SSH 복구 후 활성화 blocked

최소 유용 버전 빌드 순서

지금 바로 만들 수 있는 것부터. web app이나 DB는 나중.

Studio Skeleton

`obsidian/`, `reviews/`, `asset_pipeline/`, `runs/`, `tools/channelctl` 생성

Working Commands

`status`, `open`, `capture`, `feedback new`, `unity check`, `agent check`, `gdx probe`

Obsidian Templates

feature spec, asset brief, feedback item, session summary, sound brief, optimization note

Feedback Loop

Unity 확인 → 캡처 → 주석 → 수정 작업 → 검증 → 세션 요약

Asset Factory

2D/3D 생성 결과를 Blender cleanup과 Unity import로 이어붙임

Dashboard

latest status, open feedback, active assets, latest screenshot, next workflow 표시

현재 차단점과 금지사항

프로젝트를 빠르게 만들기 위해 지금은 운영툴을 작게 유지한다.

Blocked

gdx1 SSH 인증

네트워크와 포트는 살아 있음. 계정/키 등록이 필요.

Avoid

멀티 에이전트 동시 수정

한 브랜치 한 writer. 리뷰어는 문서/피드백만 남김.

Keep

Unity가 제품

Studio 도구는 마찰을 줄이는 보조 시스템. 제품이 되면 안 됨.
