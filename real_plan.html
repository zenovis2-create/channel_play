실행 로드맵

촬영 가능한 MVP까지 가는 단계. 각 단계는 검증 가능한 결과물로 끝남.

0단계

제작 환경 준비

Git, Git LFS, Unity, 기본 폴더, gdx1 접속 복구. 빈 씬 실행과 commit 가능 상태.

1단계

조작과 폐교 맵

플레이어 이동, 카메라, 이름표, 팀 색상, OBS 5분 녹화 가능 상태.

2단계

포인트·상점·아이템 1개

게임쇼 핵심 루프 첫 구현. 구매, 사용, 로그 기록까지 확인.

3단계

아이템 3종

진실의 펜, 위치 스캐너, 도플갱어 시약으로 심리전 최소 세트 완성.

4단계

2인 멀티플레이

스폰, 위치, 이름표, 포인트, 아이템 동기화를 서버 기준으로 검증.

5단계

4~8인 테스트

로비, 팀 배정, 미션, 타이머, 생존/탈락 상태. 30분 세션 가능성 확인.

6단계

운영자 모드

포인트·아이템·팀·공지·타이머·관전 카메라를 운영자가 제어.

7단계

방송 UI

점수판, 생존자, 남은 시간, 현재 미션, OBS용 화면 구성.

8단계

첫 파일럿 촬영

실제 참가자 4~8명으로 원본 녹화와 게임 로그 확보.

자동화 계획

수동 확인이 아니라 빌드, 실행, 로그 회수를 재현 가능하게 고정.

Mac Studio scripts

  • `scripts/check_local_env.sh`
  • `scripts/build_mac_client.sh`
  • `scripts/build_linux_server.sh`
  • `scripts/run_local_host_test.sh`
  • `scripts/collect_logs.sh`

gdx1 scripts

  • `scripts/gdx1_sync_build.sh`
  • `scripts/gdx1_run_server.sh`
  • `scripts/gdx1_run_bots.sh`
  • `scripts/gdx1_collect_logs.sh`
  • 4~8 bot, 10~30분 유지, 로그 회수

주요 리스크

차단점은 작게 만들고, Mac Studio fallback을 항상 유지.

High

gdx1 일시 단절

사용 가능 전제로 설계하되, 접속 실패 시 Mac Studio Host/Client fallback으로 진행.

Medium

Unity 미설치

Unity Hub와 LTS Editor 설치가 0단계 최우선. 설치 전 개발 착수 불가.

Medium

Git 저장소 없음

Unity 프로젝트 생성 전 Git 초기화, Git LFS, Unity gitignore 적용.

Controlled

멀티플레이 과확장

계정, 매치메이킹, 클라우드는 파일럿 후. 먼저 Host/Client와 운영자 기능 검증.

바로 다음 액션

오늘 처리할 순서. 이 순서가 끝나야 Unity 제작이 실제 작업으로 전환됨.

gdx1 접속 복구 `ssh gdx1 hostname`, Tailscale, SSH, 방화벽 확인
Git 저장소 초기화 Unity 생성 전 버전 관리 기준 확정
Git LFS 설치 대용량 에셋과 바이너리 파일 관리 준비
Unity Hub / Editor 설치 Unity 6 LTS 또는 최신 LTS 기준
Unity 프로젝트 생성 URP 기반, `_Project` 폴더 구조 생성
기본 씬 생성 `Boot`, `School_MVP`, `OperatorView` 우선
gdx1 headless server 목표 확인 Mac Studio client가 gdx1 server에 접속하는 첫 통합 목표
OBS 5분 녹화 목표 고정 첫 개발 목표는 움직임이 아니라 촬영 가능한 장면

1차 MVP 완료 기준

이 항목들이 모두 통과해야 파일럿 촬영으로 이동.

  1. 4~8명 접속
  2. 운영자 1명 접속
  3. 포인트 지급/차감 가능
  4. 상점 구매 가능
  5. 아이템 3개 사용 가능
  6. 제한시간 작동
  7. 생존/탈락 상태 표시
  8. 운영자 관전 가능
  9. 방송용 UI 표시
  10. 30분 플레이 가능
  11. 게임 로그 저장
  12. OBS 녹화본 생성
